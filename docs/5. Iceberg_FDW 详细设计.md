# GaussVector - Iceberg FDW 层详细设计

> **SDK 对齐说明（2026-06-13 修订）**：本项目已由 Java SDK 转向 **iceberg-rust**。其唯一取 Arrow 路径 `to_arrow()` 为列式读取，且**默认**做行级精确过滤（`RowFilter`）与 MOR（position/equality delete），与 Java「向量化快路径只裁剪、不精确过滤、不 MOR」不同（依据《Iceberg-SDK-能力核实（临时）》与 iceberg-rust 源码 commit `427670d5`）。原文按 Java 假设写的「下推仅 I/O 裁剪、谓词必须节点 qual 兜底 recheck」「MOR 本期不支持」等结论已按 rust 更新，下文相关处以 **（rust 对齐）** 标注。投影/谓词一律按**顶层列名**传入 SDK。
>
> 本文与 FDW 组全局文档《iceberg-fdw-managed-fullscan-implementation-design.md》（commit *Align FDW scan design with rust SDK*）口径对齐；如仍有不统一项以该全局文档为准，并在本文就近以 **（按全局文档）** 标注。

## 1. 设计背景

### 1.1 设计目标与范围

实现 Iceberg 外表只读扫描的 FDW 层：执行器查询外表时，规划回调解析 Catalog 得到表身份与数据文件描述，生成 `ForeignScan` 计划节点；执行回调据此扫描并逐行返回元组。

| 项目 | 范围 |
| --- | --- |
| 形态 | 仅行式 tuple 路径 |
| 取数 SDK | **iceberg-rust**，经 C 桥接层调用 `to_arrow()` 列式路径（行级精确过滤 + MOR 默认生效） |
| 逻辑结构 | 二级命名 `namespace.table_name`，不支持多级嵌套 namespace |
| Catalog 后端 | `pg_native`（GaussVector `iceberg_catalog.*` 元信息表） |
| 回调函数 | `GetForeignRelSize`、`GetForeignPaths`、`GetForeignPlan`、`BeginForeignScan`、`IterateForeignScan`、`EndForeignScan`、`ReScanForeignScan`、`ExplainForeignScan` |

### 1.2 整体架构

FDW 层分规划期与执行期两条回调路径，二者不共享内存，唯一通道是挂在 `ForeignScan` 节点上、可序列化的 `fdw_private` List；规划三回调之间另用 `RelOptInfo.fdw_private`（一个 `void*` 暂存槽）传递表身份等中间态，二者同名而不同字段，后者不随计划序列化。执行期把该 List 解码、落地为执行态结构 `IcebergScanState`（存于 `ForeignScanState.fdw_state`）。规划期读 Catalog 元信息表，执行期由 reader 读对象存储中的数据文件。

```
SQL / 执行器：SELECT ... FROM <iceberg 外表>
  │
  ├─ 规划期回调
  │    GetForeignRelSize    估行（total_records × 选择率）
  │    GetForeignPaths      生成访问路径
  │    GetForeignPlan       解析 Catalog → 拆分谓词(remote/local) → 编码 fdw_private → 生成 ForeignScan
  │      │
  │      └─→ Catalog 解析（只读，heap/systable 点查）：iceberg_catalog.tables_internal / snapshots
  │            （当前指针 + snapshot 摘要，全为普通列；不含 manifest/data 明细）
  │
  │   fdw_private（scanEntry + 投影列 + 下推 filter）随 ForeignScan 节点传给执行期
  │   可下推谓词转 iceberg-rust Predicate 后在 SDK Arrow 路径行级精确过滤
  │   （rust 对齐）首期仍可把 remote+local 全部谓词保留为节点 qual 作防御性 recheck，
  │    但已成功下推的顶层标量谓词正确性不再依赖该 recheck（详见 §3.3、§3.5）
  │
  └─ 执行期回调
       BeginForeignScan     解码 fdw_private → 建 IcebergScanState（挂 node->fdw_state）；组装 storage_config；iceberg_scan_open（投影/过滤/凭证下推）
       IterateForeignScan   iceberg_scan_next 取 Arrow 批 → 转 HeapTuple → 逐行返回
       EndForeignScan       iceberg_scan_close，释放内存上下文
         │
         └─→ reader（IcebergScan，下半层 / fdw-iceberg-sdk-arrow_design.md）：metadata.json → manifest → data file → Parquet → Arrow
               （iceberg-rust `to_arrow()` 列式路径；position/equality delete 由 SDK 默认应用）
```

### 1.3 FdwRoutine 注册

handler 用 `makeNode(FdwRoutine)` 创建结构（未赋值字段自动为 `NULL`），逐字段赋回调指针：

| 字段 | 赋值 | 说明 |
| --- | --- | --- |
| `GetForeignRelSize` / `GetForeignPaths` / `GetForeignPlan` | 是 | 规划三回调（§3.1–3.3） |
| `BeginForeignScan` / `IterateForeignScan` / `EndForeignScan` | 是 | 扫描生命周期（§3.4–3.6） |
| `ReScanForeignScan` / `ExplainForeignScan` | 是（占位） | 注册占位，本期不展开（§3.6） |

### 1.4 OPTIONS 配置与校验

每张外表归属一个 SERVER，SERVER 绑定 FOREIGN DATA WRAPPER。catalog 后端配置为 server 下所有外表共享，置于 server；表身份逐表声明，置于 table。

| 层级 | OPTION | 用途 |
| --- | --- | --- |
| server | `catalog_kind` | Catalog 后端 |
| server | `catalog_schema` | 元信息表所在 schema |
| table | `namespace` | Iceberg 表所属 namespace（对应 DB schema） |
| table | `table_name` | Iceberg 表名 |
| server / user mapping | 存储访问配置（endpoint/region/密钥等） | 组装为 `storage_config` 传 reader（§4.3） |

## 2. 数据结构

同一份扫描所需信息，在 DDL、规划、执行三阶段由不同结构承载，后一阶段的数据由前一阶段生成；规划回调与执行回调不共享内存，只能通过 `fdw_private` 传递。

| 阶段 | 数据 | 存储位置 | 生成者 | 使用者 |
| --- | --- | --- | --- | --- |
| DDL | OPTIONS | 系统目录（`pg_foreign_table` / `pg_foreign_server`） | `CREATE ... OPTIONS`，validator 校验 | 规划回调 |
| 规划 | `fdw_private` | `ForeignScan` 计划节点（可 `copyObject`） | `GetForeignPlan` | 执行回调 |
| 执行 | `IcebergScanState` | 进程内存（`node->fdw_state`） | `BeginForeignScan` | `Iterate` / `End` |

### 2.1 `fdw_private` 编码

`ForeignScan` 节点上的 `fdw_private` 是规划 → 执行的唯一通道，按固定枚举顺序装配成一个可序列化 `List`（与 §3.1 里 `RelOptInfo.fdw_private` 那个仅在规划三回调间传递表身份的 `void*` 暂存槽同名而不同字段）；它在 `GetForeignPlan` 一次装配、经 `make_foreignscan` 写入节点，此后只读、随计划序列化，执行期按枚举位解码：

```c
enum IcebergScanPrivateIndex {
    IcebergPrivScanEntry,       /* 扫描入口：metadata_location + snapshot_id + schema_id */
    IcebergPrivRetrievedAttrs,  /* 投影列属性号 List<int>（列下推） */
    IcebergPrivPushedFilter,    /* 下推过滤：字段以顶层列名引用，桥接层构造为 iceberg-rust Predicate；SDK 行级精确过滤 */
};
```

- **`IcebergPrivScanEntry`**：本次扫描的入口指针（§2.3）；不含文件列表，文件规划由 reader 内部完成。
- **`IcebergPrivRetrievedAttrs`**：需读取回填的外表列属性号（attno），即**列下推**；`Begin` 经 `get_attname` 译成 Iceberg 列名传给 reader（§3.4）。
- **`IcebergPrivPushedFilter`**：可下推谓词的序列化形式（字段以**顶层列名**引用，协议见 §4.3）；为空表示无下推。`Begin` 解码后由桥接层构造为 iceberg-rust `Predicate`。**（rust 对齐）** `to_arrow()` 路径对该 `Predicate` 做**行级精确过滤**（`RowFilter`），非仅 I/O 裁剪；故就正确性而言已成功下推的顶层标量谓词**无须**节点 qual 兜底。首期是否仍把全部谓词保留为节点 qual 复核属**防御策略**（§3.3、§3.5）；桥接层无法转 `Predicate` 的谓词本就留作 local qual。

### 2.2 `IcebergScanState` 执行态

```c
typedef struct IcebergScanState {
    Relation        rel;
    TupleDesc       tupdesc;
    List           *retrievedAttrs;  /* 投影 attno，解码自 fdw_private（scanEntry/filter 解码后即传 reader，不在执行态留存） */

    void           *reader;          /* IcebergReaderState*（fdw-iceberg-sdk-arrow_design.md §3.3.2）：{ IcebergScan*; Converter* } */

    /* 批取数缓冲（逐批读入、逐行返回） */
    HeapTuple      *tuples;
    int             num_tuples;
    int             next_tuple;
    bool            eof_reached;

    MemoryContext   batch_cxt;       /* 每批 reset 一次性回收 */
    MemoryContext   temp_cxt;
} IcebergScanState;
```

`IcebergScanState` 是本层执行态的具体类型，由 `Begin` 分配后存入 `ForeignScanState` 的通用 `void* fdw_state` 槽（`node->fdw_state`），其后 `Iterate` / `End` 经 `node->fdw_state` 取回；其内容由 `fdw_private` 解码而来，再补入 reader 句柄、批缓冲与内存上下文等执行期资源。`retrievedAttrs` 由 `Begin` 从 `fdw_private` 解码填入；`rel`/`tupdesc`、`reader`（=`IcebergReaderState*`，内含 `IcebergScan` 句柄与 Arrow→Datum 转换函数表 `converters`，于 `Begin` 按 `iceberg_scan_open` 返回的 `ArrowSchema` 建好）、批缓冲字段（沿用 postgres_fdw / pg_lake 同名字段）、`*_cxt` 由 `Begin` 现场建立；缓冲字段在 `Iterate`/`fetch_more_data` 间推进重置，`*_cxt` 在 `End` 释放。

### 2.3 扫描入口与文件规划

**ScanEntry（FDW 编码）**：`{ metadata_location, snapshot_id, schema_id }`，均取自 `tables_internal` 普通列。`fdw_private` 只带此入口，不带文件列表。（**按全局文档**：其 `IcebergFdwScanEntry` 另含 `relid`、`table_uuid`，供执行期 delta 表查找与 catalog 一致性诊断；本文 delta 为预留路径（§4.4），落地 delta 时按全局文档补 `relid`/`table_uuid`。）

**FileScanTask（reader 内部）**：单个数据文件的扫描描述（文件路径、格式等）。Catalog 只缓存当前指针与 snapshot 摘要、**不缓存 manifest / data file 明细**（见 §4），故文件规划必须在运行期读对象存储产出：

```
snapshots.manifest_list（已缓存普通列）→ manifest → data file → FileScanTask
```

**产出时机（执行期）**：文件规划在 `BeginForeignScan` 调 `iceberg_scan_open` 时由 reader 内部完成——FDW 传入 ScanEntry（`metadata_location`(+`snapshot_id`)）、`storage_config`、投影 `columns`、下推 `filter`；reader 据 `metadata_location` 解析 `metadata.json → manifest list → manifest → data file` 产出 FileScanTask 并读取（命中 `snapshots.manifest_list` 缓存可免读 `metadata.json`）。规划期不访问对象存储。

## 3. 回调详细设计

执行次序与数据流：规划期 `GetForeignRelSize → GetForeignPaths → GetForeignPlan`（产出 `fdw_private`）；执行期 `Begin → Iterate（重复）→ End`。各回调输入由规划器/执行器传入，对元信息表与对象存储的访问在回调内经 heap/systable 点查或 reader 发起，统一列在「依赖」项下。

| 回调 | 触发时机 | 职责 |
| --- | --- | --- |
| `GetForeignRelSize` | 规划 | 估行，写 `baserel->rows` |
| `GetForeignPaths` | 规划 | 登记全表扫描路径 |
| `GetForeignPlan` | 规划 | 拆分谓词、解析 Catalog，编码 `fdw_private`，生成节点 |
| `BeginForeignScan` | 执行开始 | 解码 `fdw_private`，建扫描态与内存上下文 |
| `IterateForeignScan` | 逐行（热路径） | 批缓存状态机，逐行返回 tuple |
| `EndForeignScan` | 执行结束 | 释放 reader 与内存上下文 |

### 3.1 `GetForeignRelSize`

估计外表返回行数，供优化器比较路径代价。

- **输入**：`PlannerInfo *root`（规划上下文）、`RelOptInfo *baserel`（外表关系优化信息）、`Oid foreigntableid`（外表 OID）。
- **输出**：无返回值；写 `baserel->rows`（估计返回行数）。平均行宽由优化器按列类型给出。

```
void GetForeignRelSize(root, baserel, foreigntableid):
    // 1. 取表身份：table OPTIONS(namespace/table_name) + server OPTIONS(catalog 配置)
    ident = IcebergGetTableIdentity(foreigntableid)
    baserel->fdw_private = ident                    // RelOptInfo 的规划期 void* 暂存槽，供 Paths/Plan 复用；非节点上的 fdw_private List

    // 2. heap 点查 snapshots，取缓存 summary 的 total-records 作表基数
    rows = IcebergGetTableCardinality(ident)

    // 3. 按谓词选择率折算
    sel = clauselist_selectivity(root, baserel->baserestrictinfo, ...) // 当前为硬编码
    baserel->rows = rows * sel
```

- **依赖**：表基数取自 `snapshots` 缓存 summary 的 `total-records`（上游缓存 summary）。

### 3.2 `GetForeignPaths`

生成外表访问路径并登记到候选集。

- **输入**：`PlannerInfo *root`、`RelOptInfo *baserel`、`Oid foreigntableid`。
- **输出**：无返回值；向 `baserel->pathlist` 登记一条全表扫描 `ForeignPath`。
- **依赖**：无；不访问元信息表或对象存储（表身份已在 §3.1 暂存于 `baserel->fdw_private`）。

```
void GetForeignPaths(root, baserel, foreigntableid):
    path = create_foreignscan_path(root, baserel, ...)   // 单条全表扫描路径，无 pathkeys
    add_path(baserel, path)
```

### 3.3 `GetForeignPlan`

将选中路径固化为 `ForeignScan` 节点：拆分谓词、解析 Catalog、编码 `fdw_private`。

- **输入**：`PlannerInfo *root`、`RelOptInfo *baserel`、`Oid foreigntableid`、`ForeignPath *best_path`（选中路径）、`List *tlist`（目标列）、`List *scan_clauses`（扫描谓词，一组 `RestrictInfo`）、`Plan *outer_plan`（openGauss 特有）。
- **输出**：`ForeignScan *` 计划节点（`make_foreignscan` 构造，`fdw_private` 为第 5 参）。

```
ForeignScan *GetForeignPlan(root, baserel, foreigntableid,
                            best_path, tlist, scan_clauses, outer_plan):
    // 1. 拆分谓词：可下推(remote) / 本地(local)，依据 is_foreign_expr（Iceberg 可表达性）
    (remote_conds, local_conds) = classifyConditions(scan_clauses)  // remote = 可转 iceberg-rust Predicate 者
    pushedFilter = SerializeFilter(remote_conds)                // 字段按顶层列名；桥接层据此建 Predicate（§4.3）
    // （rust 对齐）下推谓词在 SDK 内行级精确过滤；首期 scan_quals 仍取全部谓词作防御性 recheck，
    //  后续确认桥接覆盖完整后可只保留未下推残差 local_conds
    scan_quals   = extract_actual_clauses(scan_clauses, false)

    // 2. 解析 Catalog：heap 查询 tables_internal，无对象存储访问
    ident = baserel->fdw_private                                // 复用 RelSize 阶段
    meta  = IcebergCatalogResolveTable(ident)                   // §4.1：systable 点查

    // 3. 算投影列 attno（reltarget + local_conds 用到的列，pull_varattnos）
    retrievedAttrs = BuildRetrievedAttrs(baserel)

    // 4. 构造扫描入口（文件规划在执行期由 reader 完成，规划期不读对象存储）
    scanEntry = {meta.metadata_location, meta.current_snapshot_id, meta.current_schema_id}

    // 5. 装配节点上的 fdw_private List（§2.1 枚举序）；经 make_foreignscan 写入 ForeignScan 节点，此后只读、随计划序列化
    fdw_private = IcebergBuildScanPrivate(scanEntry, retrievedAttrs, pushedFilter)   // = list_make3(scanEntry, retrievedAttrs, pushedFilter)
    return make_foreignscan(tlist, scan_quals, scanrelid, NIL/*fdw_exprs*/, fdw_private, ...)   // qual=全部谓词，供 ExecScan recheck

```

- **依赖**：步骤 2 heap+index 点查 `iceberg_catalog.tables_internal`，取 `metadata_location` / `table_location` / `current_schema_id` / `current_snapshot_id` / `table_uuid`。其中`current_schema_id` / `current_snapshot_id` 随计划下传给 reader（据此读 schema、定位 snapshot、PlanFiles）。
- **谓词下推与 recheck（rust 对齐）**：按 `is_foreign_expr`（参考 postgres_fdw）判定每个谓词能否表达为 iceberg-rust `Predicate`，可表达者（remote 简单谓词：AND 合取的 `col OP const`、`col IS [NOT] NULL`）序列化进 `pushedFilter`、由桥接层构造为 `Predicate` 下推。iceberg-rust 的 `to_arrow()` 路径对该 `Predicate` **行级精确过滤**（`RowFilter`，依据《Iceberg-SDK-能力核实（临时）》§2.1），并非仅 I/O 裁剪；因此就正确性而言已成功下推的顶层标量谓词**不再必须**保留为节点 qual。**首期取舍**：为降低初期桥接覆盖不全的风险，仍把全部 `scan_clauses` 放进节点 qual 作**防御性** recheck（机制见 §3.5）；待桥接谓词覆盖确认后可只保留未下推残差（与 postgres_fdw「下推后不再本地过滤」一致）。注：`col IN (const…)`、`<>` 等本期保留本地、不下推（与全局文档 §6.2 一致），详见 §4.3。

### 3.4 `BeginForeignScan`

建立执行态，解码 `fdw_private`，初始化内存上下文与批游标。

- **输入**：`ForeignScanState *node`（扫描执行态节点）、`int eflags`（执行标志位）。
- **输出**：无返回值；把新建的 `IcebergScanState` 存入 `ForeignScanState` 的通用 `fdw_state` 槽（`node->fdw_state`），`EXEC_FLAG_EXPLAIN_ONLY` 时保持 `NULL`。

```
void BeginForeignScan(ForeignScanState *node, int eflags):
    if eflags & EXEC_FLAG_EXPLAIN_ONLY:
        return                                       // fdw_state 保持 NULL

    IcebergScanState *festate = palloc0(sizeof(IcebergScanState))   // node->fdw_state 的具体类型
    festate->rel     = node->ss.ss_currentRelation
    festate->tupdesc = RelationGetDescr(festate->rel)

    // 解码节点上的 fdw_private List（顺序严格对应 §2.1）
    plan                    = (ForeignScan *) node->ss.ps.plan
    scanEntry               = list_nth(plan->fdw_private, IcebergPrivScanEntry)     // {metadata_location, snapshot_id, schema_id}
    festate->retrievedAttrs = list_nth(plan->fdw_private, IcebergPrivRetrievedAttrs)
    pushedFilter            = list_nth(plan->fdw_private, IcebergPrivPushedFilter)  // 序列化谓词（字段为顶层列名），可空；下方由桥接层转 Predicate

    // attno → Iceberg 列名（get_attname）；从 OPTIONS 组装存储凭证
    columns        = AttnosToColumnNames(festate->tupdesc, festate->retrievedAttrs)
    storage_config = BuildStorageConfig(server_options, user_mapping_options)       // §1.4/§4.3

    festate->batch_cxt = AllocSetContextCreate(...)
    festate->temp_cxt  = AllocSetContextCreate(...)
    festate->tuples = NULL; festate->num_tuples = 0
    festate->next_tuple = 0; festate->eof_reached = false

    // reader 据入口内部 PlanFiles，施加列投影、I/O 裁剪与行级过滤（iceberg-rust to_arrow()；position/equality delete 默认应用）
    reader = palloc0(IcebergReaderState)
    reader->scan = iceberg_scan_open(festate->batch_cxt,
                                     scanEntry.metadata_location, storage_config,
                                     columns, n_columns, pushedFilter, &out_schema) // 下半层，fdw-iceberg-sdk-arrow_design.md §3.2.3
    reader->converters = BuildConverters(out_schema)   // Arrow→Datum 函数表，fdw-iceberg-sdk-arrow_design.md §3.3.4
    festate->reader = reader
    node->fdw_state = festate                        // 存入 ForeignScanState 的通用 void* 槽；Iterate/End 经此取回
```

- **依赖**：解码顺序与 §2.1 编码顺序严格对应；attno→列名经 `get_attname`；本回调不访问 Catalog 或对象存储（对象存储访问在 reader 内）。
- **说明**：`iceberg_scan_open` 属下半层（fdw-iceberg-sdk-arrow_design.md）；文件规划、列投影/行过滤、delete 应用均在 reader 内；`storage_config` 由 server/user-mapping OPTIONS 组装（§4.3）。

### 3.5 `IterateForeignScan`

每次返回一行；数据耗尽返回空槽表示结束。PG 每次只取一行，reader 一次返回整批，故内部维护批缓存状态机。

- **输入**：`ForeignScanState *node`。
- **输出**：`TupleTableSlot *`；含一行数据的扫描槽，或经 `ExecClearTuple` 清空的空槽（结束）。
- **依赖**：无上游元信息表访问；取数经 reader（对象存储）发起。

```
TupleTableSlot *IterateForeignScan(ForeignScanState *node):
    festate = node->fdw_state
    if festate == NULL:                              // EXPLAIN-only
        return ExecClearTuple(slot)

    // 批已耗尽则拉下一批
    if festate->next_tuple >= festate->num_tuples:
        fetch_more_data(festate)
        if festate->num_tuples == 0:                 // reader 结束
            return ExecClearTuple(slot)

    // qual 不在此施加：每行交由 ExecScan 外壳调 ExecQual 复核（§3.3 的全部谓词）
    return ExecStoreTuple(festate->tuples[festate->next_tuple++], slot)

fetch_more_data(festate):
    reader = festate->reader                          // IcebergReaderState
    MemoryContextReset(festate->batch_cxt)            // 回收上一批
    // reader 返回 Arrow 批（已投影/已过滤）；FDW 在此转 HeapTuple，fdw-iceberg-sdk-arrow_design.md §3.3.3
    nrows = iceberg_scan_next(reader->scan, &arrow_array, &arrow_schema)   // 下半层
    festate->num_tuples = nrows
    if nrows > 0:
        festate->tuples = ArrowBatchToHeapTuples(arrow_array, reader->converters,
                                                 festate->tupdesc, festate->batch_cxt)
    festate->next_tuple = 0
```

- **依赖**：`iceberg_scan_next` 属下半层（fdw-iceberg-sdk-arrow_design.md：C 桥接→iceberg-rust `to_arrow()`→Parquet→Arrow），返回 `int nrows`(0=结束) + 出参 `ArrowArray*`/`ArrowSchema*`；FDW 用 `converters` 转 HeapTuple 入 `batch_cxt`。
- **说明**：列投影、I/O 裁剪与行级过滤在 `open` 时已绑定、由 reader 读取时施加（§3.4、§4.3），本回调只负责 Arrow→tuple 转换与逐行吐出，自身不做过滤。
- **谓词复核（rust 对齐）**：iceberg-rust 下推谓词已在 SDK 内行级精确过滤，故节点 qual 复核对**已下推的顶层标量谓词是防御层而非必需**；其主要必要性在于承接**未下推的 local 谓词**。复核机制本身是 ForeignScan 节点执行流程的固有环节：执行器以 `ExecForeignScan → ExecScan` 驱动本节点，`ExecScan` 每从 `Iterate` 取得一行即按 `node->ss.ps.qual` 调 `ExecQual` 校验，命中返回、否则丢弃续取。`ExecQual` 是执行器现成的函数，谓词在 `GetForeignPlan` 经 `make_foreignscan` 写入节点 qual 后即自动生效，FDW 无需在 `Iterate` 内自行调用。首期可保留全部谓词复核以防御桥接覆盖不全（见 §3.3）。

### 3.6 `EndForeignScan`

输入 `ForeignScanState *node`，无返回值，不访问上游元信息表；结束扫描、释放 reader 与内存上下文。

```
void EndForeignScan(ForeignScanState *node):
    festate = node->fdw_state
    if festate == NULL:                              // EXPLAIN-only
        return
    iceberg_scan_close(festate->reader->scan)        // 下半层
    MemoryContextDelete(festate->batch_cxt)
    MemoryContextDelete(festate->temp_cxt)
```

`ReScanForeignScan` 与 `ExplainForeignScan` 均作占位：FdwRoutine 要求注册，本期不展开。ReScan 的最小语义是关旧重开，使后续 `Iterate` 从首个 task 重读；Explain 仅注册桩，不向 EXPLAIN 输出附加项。两者占位不影响六个核心回调；EXPLAIN-only 路径仍由 `Begin` 早退（`fdw_state == NULL`）保护。

> **（按全局文档）**：FDW 组全局文档 §7.9/§7.11 已把 `ReScanForeignScan` 实现为「关闭并重开 SDK / delta 扫描句柄」，并为 `ExplainForeignScan` 定义了完整输出（`Iceberg Table/UUID/Metadata/Snapshot/Schema ID`、`Projection Columns`、`SDK Filter`、`Filter Mode: rust predicate exact; local recheck enabled/disabled`、`Delta Scan`、`MOR Support: provided by rust SDK`）。本文按本期范围仍作占位，落地时应采用全局文档的 ReScan 行为与 EXPLAIN 输出口径。

## 4. 上下游边界

FDW 层解析入口，将外表身份解析为指针与 snapshot 摘要；列结构解析与文件读取交给 reader。

### 4.1 `IcebergCatalogResolveTable`（pg_native）

把外表身份 `(namespace, table_name)` 解析为权威 `metadata.json` 路径与当前 schema / snapshot 指针，对应 Iceberg `loadTable` 的只读部分，仅依赖 `iceberg_catalog.tables_internal`。

- **入口**：`IcebergGetTableIdentity(foreigntableid)` —— 入参是 PG 外表 relation 的 OID：`GetForeignTable(oid)->options` 取 `namespace`/`table_name`，`GetForeignServer(serverid)->options` 取 `catalog_kind`/`catalog_schema`，即把 PG OID 翻译为 catalog 主键 `(namespace, table_name)`。
- **解析**：

```
rel  = heap_open(<catalog_schema>.tables_internal 的 relid, AccessShareLock)
ScanKeyInit(&k[0], Anum_namespace,  BTEqualStrategyNumber, F_TEXTEQ, ns)
ScanKeyInit(&k[1], Anum_table_name, BTEqualStrategyNumber, F_TEXTEQ, tbl)
scan = systable_beginscan(rel, tables_internal_pkey, /*indexOK*/true, GetActiveSnapshot(), 2, k)
tup  = systable_getnext(scan)          // 命中 0 行 → ereport(NoSuchTable)
... = heap_getattr(tup, ...)           // metadata_location/table_location/current_schema_id/current_snapshot_id/table_uuid
systable_endscan(scan); heap_close(rel, AccessShareLock)
```
  - 表/索引 OID 由 `(catalog_schema, 'tables_internal')` 经 `get_relname_relid` 解析一次（可缓存）；快照取 `GetActiveSnapshot()`。
  - 结果用调用方上下文拷贝（`pstrdup`），`metadata_location` 为空时报错。
- **结果**：`{ metadata_location, table_location, current_schema_id, current_snapshot_id, table_uuid }`。

### 4.2 上游元数据表依赖

FDW 层规划期消费的元数据，及其在 Catalog 表中的形态与取用方式（直接回答「字段是否已在表结构中 / 是否 JSON 封装 / 是否需规划期访问对象存储」）：

| 数据 | Catalog 表中的形态 | 取用方式 | 使用方 |
| --- | --- | --- | --- |
| `metadata_location` | `tables_internal` 普通列 | 查 `tables_internal` | `GetForeignPlan` 编入 scanEntry（reader PlanFiles 入口） |
| `current_snapshot_id` | `tables_internal` 普通列 | 查 `tables_internal` | `GetForeignPlan` 定位待扫描 snapshot；`RelSize` 定位估行 snapshot |
| `current_schema_id` | `tables_internal` 普通列 | 查 `tables_internal` | `GetForeignPlan` 下传 reader（FDW 不解释列结构） |
| `table_uuid` | `tables_internal` 普通列 | 查 `tables_internal`                                                 | 关联 `snapshots` / `table_schemas` 行 |
| `manifest_list` | `snapshots` 普通列 | reader 读（命中则免读 `metadata.json`） | reader 的 PlanFiles（执行期） |
| 表记录数 `total-records` | `snapshots` 缓存 summary（上游） | heap 点查 `snapshots`（`table_uuid`+`current_snapshot_id`） | `GetForeignRelSize` |
| `schema_json` | `table_schemas` **JSONB**（catalog 镜像；权威 schema 以 `metadata.json` 为准） | **（rust 对齐）** reader 经 `metadata_location` 读 `metadata.json` 内嵌 schema（iceberg-rust `StaticTable` 不读 catalog `table_schemas`） | reader 列对应（§4.3） |
| manifest / data / delete file 明细 | **未缓存**（设计排除持久化） | reader 在 `Begin` 内 PlanFiles 读对象存储 | reader |

**边界**：manifest 与数据文件明细不在 Catalog 内，须运行期由 reader 读对象存储解析（含 `metadata.json` 内嵌 schema 与 manifest）。

### 4.3 下游 reader 接口与边界

reader 是 FDW 层的取数后端（`IcebergScan`，fdw-iceberg-sdk-arrow_design.md），对 FDW 层是不透明句柄；接口即 fdw-iceberg-sdk-arrow_design.md §3.2.1 的 open/next/close。`open`/`next` 均以入参发起，结果经返回值与出参带回；表身份不下传——`open` 传入的是规划期已解析的完整 `metadata_location`。逐参数清单与 SDK 内部下推能力的参数另见《oG-Iceberg-FDW SDK 接口与下推参数》。

| 接口 | 方向 | 参数 / 返回（FDW 经桥接层传下 · SDK 返回） |
| --- | --- | --- |
| `iceberg_scan_open` | 入 | `cxt` 内存上下文 · `metadata_path` 完整 `metadata_location`（规划期 Catalog 解析，非表身份）· `storage_config` 凭证 · `columns`/`n_columns` 投影**列名**（非 field_id）· `filter` 下推谓词（可空，字段按顶层列名引用，桥接层转 iceberg-rust `Predicate`） |
| `iceberg_scan_open` | 回 | `IcebergScan*` reader 指针 + 出参 `out_schema`（`ArrowSchema*`，建 converters） |
| `iceberg_scan_next` | 入 | `scan` reader 指针 |
| `iceberg_scan_next` | 回 | `int nrows`(0=结束) + 出参 `out_array`（`ArrowArray*` Arrow 批）+ `out_schema` |
| `iceberg_scan_close` | 入 | `scan` reader 指针 |
| `iceberg_scan_close` | 回 | 无返回值；释放 reader 与扫描资源 |

**边界**：FDW 层只传递 open_reader 所需元数据，以下完全由 reader 实现：

- 读 `metadata.json` / `manifest` / `*.parquet`（PlanFiles）；
- 据 `metadata_location` 读 `metadata.json` 内嵌 schema，按 `field_id` 对应文件列到表列（schema evolution）并做类型映射（**rust 对齐**：iceberg-rust 内部以 field_id 经 `ProjectionMask` 投影；schema 取自 metadata 而非 catalog `table_schemas`）；
- 应用 delete file（V2 MOR）：**（rust 对齐）由 iceberg-rust `to_arrow()` 默认应用**位置删除（`RowSelection`）与等值删除（转 `Predicate` 与 filter 取 AND）；FDW 层不自行实现 MOR 合并。注意这与 §4.4 的 openGauss **delta 表**不是同一概念。

**下推 filter 接口（`IcebergPrivPushedFilter`）**：

- 范围（**按全局文档 §6.2**）：AND 合取的简单谓词 `col OP const`(OP∈ =,<,<=,>,>=)、`col IS [NOT] NULL`；`varchar/text` 仅 `=`，`bpchar` 一般不下推。**`<>`、`col IN (const…)`、`LIKE`、正则、字符串范围、函数/volatile 表达式本期不下推、留本地**（注：iceberg-rust `Predicate` 支持 `is_in` 等，技术上可下推，全局文档为控初期风险暂缓，后续可放开）。仅非嵌套顶层列、可映射类型；过 `is_foreign_expr` 等价判断者才转 `Predicate`。
- 字段引用：**顶层列名**（非 field_id）。桥接层据此建 `Reference::new(name)`（`expr/term.rs`）并组合为 `Predicate`；`bind(schema)` 时解析为 field。
- 序列化格式：`fdw_private` 内可用最小 DSL `[{field, op, type, value}]`（或 Iceberg `ExpressionParser` JSON）承载，`Begin` 解码后构造 `Predicate`；该序列化**仅为计划内传输编码，非传给 SDK 的 SQL 串**。字面量带类型；NULL 用 isNull/notNull。

### 4.4 执行期取数后端的扩展边界（预留）

`IterateForeignScan` 之下的取数来源可并列，本期仅无索引 iceberg-rust SDK 一条（`iceberg_scan_*`，读对象存储已落 S3 的数据，§4.3）。后续在同一层级平行接入另两类来源，三者最终都在 `IterateForeignScan` 内汇聚为 HeapTuple 流、统一经节点 qual recheck（§3.5）返回，但取数形态不同：

| 来源 | 数据位置 | 取数形态 | 本期 |
| --- | --- | --- | --- |
| 无索引 iceberg-rust SDK | 对象存储（已落 S3 的 Iceberg 数据） | 全表扫描返回 Arrow 批，经 `ArrowBatchToHeapTuples` 转 HeapTuple | 实现 |
| 索引 SDK | 对象存储 + 向量索引 | 按候选返回 Arrow 批，同样转 HeapTuple | 预留（待索引组） |
| delta 表 | openGauss 本地存储层 | 本地行存表/视图，`tableam` 扫描直接得 HeapTuple，不经 Arrow | 预留接口 |

delta 表是 openGauss 本地存储的普通表，其原生形态即行式 HeapTuple，故 delta 路径绕过 Arrow 转换、直接产元组；iceberg-rust SDK 两类来源（无索引 / 索引）则先得 Arrow 批再转 HeapTuple。三者汇聚后的归并次序与去重（delta 与索引 / 全表之间），待索引组接口与 delta 形态明确后定。本期取数循环不假设单一来源，归并逻辑暂不展开。
